Local extensions

Build small trusted extensions without depending on De-Koi internals.

De-Koi can import local CSS and JavaScript extensions from Settings > Extensions. The supported public surface for JavaScript extensions is the marinara object passed to the extension at runtime. Treat React components, app stores, and private DOM structure as implementation details.

Import Formats

Format What De-Koi reads Initial enabled state
.css The whole file is stored as extension CSS and injected while the extension is enabled. Activated only on the importing device after its content fingerprint is recorded. Profile and remote imports do not activate it on other devices.
.js The whole file is stored as extension JavaScript and executed through the extension runtime. Installed disabled. The user must review it and grant device-local activation.
.json A record with name, optional description, optional css, and optional js. Disabled when js has runnable code; otherwise enabled.

Package Manifest

De-Koi can import a JSON extension package manifest with manifestVersion: 1. The first package slice records identity, version, compatibility, permissions, and declared UI slots; it does not grant new runtime capabilities beyond the existing CSS, JavaScript, and scoped plugin-memory APIs. 

{
  "manifestVersion": 1,
  "id": "soft-reading-mode",
  "name": "Soft Reading Mode",
  "description": "Makes long chats calmer to read.",
  "version": "1.0.0",
  "compatibility": { "deKoi": ">=1.6.0 <2.0.0" },
  "permissions": ["ui:styles", "storage:plugin-memory"],
  "ui": { "slots": ["settings", "overlay"] },
  "entrypoints": {
    "css": "body { line-height: 1.65; }",
    "js": null
  }
}

Package compatibility uses semantic version ranges. Package manifests must declare a matching compatibility.deKoi range before activation; missing, malformed, and incompatible ranges remain blocked. Profile imports never create device-local activation, even if exported metadata marks an extension available.

JavaScript extensions run as trusted page-level code. Manifest permissions limit only the helpers De-Koi puts on marinara; they are not a browser sandbox and do not prevent direct page access. Activation is stored per runtime and device and becomes stale whenever code, styles, package identity, version, or permissions change.

Runtime API

JavaScript is wrapped by the extension loader and receives one object named marinara. This object is the public API. Do not import app source files, call internal stores, or depend on private React component names.

Package extensions receive helper groups only when declared: ui:styles exposes addStyle, runtime:dom exposes DOM, timer, observer, and cleanup helpers, and storage:plugin-memory exposes storage. Settings labels other declarations unavailable until De-Koi implements them. Legacy file extensions retain the older unscoped helper surface after local activation.

API Use it for
extensionId, extensionName Stable identity and display name for the installed extension.
addStyle(css) Add a <style> tag that De-Koi removes when the extension is disabled.
addElement(parent, tag, attrs) Add a DOM element under a selector or element. Attribute values are strings; textContent and innerHTML are handled specially.
storage Scoped access to the plugin-memory collection.
on(target, event, handler) Add an event listener that is removed during extension cleanup.
setInterval(fn, ms), setTimeout(fn, ms) Schedule timers that De-Koi clears during extension cleanup.
observe(target, callback, options) Create a MutationObserver that disconnects during extension cleanup.
onCleanup(fn) Register custom teardown for anything the runtime does not already own.

Storage Scope

Extension storage is intentionally narrow. The runtime allows only the plugin-memory entity and automatically scopes records to a host-controlled namespace. A package manifest cannot choose this namespace.

  • List calls are filtered to the current extension's pluginId.
  • Create calls force pluginId to the host-assigned namespace.
  • Record IDs must stay inside the extension namespace: <extensionId>:<encoded key>.
  • Update calls cannot change id or key.

Removing an extension offers two explicit choices: retain its data for a later reinstall, or purge it. Retained records expose identity and row-count metadata in Settings, never their stored values. Reconnection requires an explicit action and an exact match for package ID, package version, and declared permissions. Altered or mismatched packages cannot claim retained data. 

const key = "panel-enabled";
const id = `${marinara.extensionId}:${encodeURIComponent(key)}`;

const existing = await marinara.storage.get("plugin-memory", id);
if (existing) {
  await marinara.storage.update("plugin-memory", id, {
    value: { enabled: true },
    schemaVersion: 1,
  });
} else {
  await marinara.storage.create("plugin-memory", {
    key,
    value: { enabled: true },
    schemaVersion: 1,
  });
}

Cleanup

Prefer runtime helpers over raw browser APIs when possible. Elements from addElement, styles from addStyle, listeners from on, timers from setInterval and setTimeout, and observers from observe are cleaned up automatically when the extension is disabled or reloaded.

Use onCleanup for custom teardown such as removing global flags, closing a third-party widget, or undoing a change that the runtime did not create for you.

Security Expectations

  • Only import JavaScript extensions from authors and files you trust.
  • Do not store secrets in extension source or extension memory.
  • Avoid global monkeypatching unless the extension fully owns the risk and cleanup.
  • Prefer stable selectors you create yourself over De-Koi's private internal DOM structure.
  • Keep extensions small and reversible so disabling them returns the page to normal.

Customization Recovery And Fonts

Theme preview starts off and runs in a sandboxed iframe. If a customization breaks the interface, open ?safe-mode=customizations to load without customization providers, deactivate the theme, and clear this device's extension activations.

Embedded De-Koi can open its managed fonts folder. Remote runtimes instead offer a validated upload for TTF, OTF, WOFF, and WOFF2 files up to 10 MiB; the server verifies filename safety, extension, and file signature. Rejected uploads remain uninstalled and Settings displays the validation error so the file can be corrected.
