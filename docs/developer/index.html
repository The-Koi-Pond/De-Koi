Start here

How this repo is put together and how to work in it without widening impact.

De-Koi is a Tauri desktop app with a hostable Rust runtime for supported remote execution. React owns the user interface, the TypeScript engine owns product behavior, and Rust owns privileged capabilities such as storage, filesystem access, provider transport, secrets, integrations, and the HTTP runtime facade.

Refactor Coverage

Shell and feature split

The app shell lives under src/app. Product surfaces live under src/features, including chat, roleplay, game, assets, libraries, settings, integrations, onboarding, and Deki-senpai.

Engine split

Engine code is separated into contracts, capabilities, pure helpers, repositories, generation building blocks, agent runtime, generation lifecycle, standalone Mari entry, and mode-owned services.

Deki-senpai path

Deki-senpai is not hidden inside chat. The shell opens features/shell/deki, which calls engine/deki, shared/api/deki-api, and the Rust deki_prompt command.

Native capability split

src-tauri is split into thin command facades plus focused crates for core helpers, security, storage, assets, LLM transport, and integrations. The same focused Rust modules back the embedded Tauri commands and hostable HTTP dispatch.

Rules That Keep Changes Small

One owner

Every behavior belongs to React UI, the TypeScript engine, or Rust capability code. Do not duplicate ownership across layers.

Mode separation

Chat, roleplay, and game are sibling product modes. They share lower-layer primitives, not each other's orchestration.

Explicit imports

Import owner modules and public feature APIs. Avoid deep imports into another feature's private files, and keep embedded Tauri or hostable HTTP calls behind shared API wrappers.

Root fixes

Fix broken contracts or owners. Do not add fake success paths, broad fallbacks, or UI-only patches.

Before a change: identify the owner, expected impact area, dependent callers, and checks. If a shared layer changes, verify every mode or feature that depends on that layer.
